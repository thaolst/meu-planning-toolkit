# MEU Gap Analysis - Resumption

Use this when returning to a gap analysis already started in a previous session, or when updating the analysis mid-quarter with new data.

## When to use

- Monthly check-in against MEU target
- After a major campaign runs and you have results
- When leadership asks for a mid-period update

## Prompt

```
We are continuing a MEU gap analysis. Here is the current state:

Previous diagnosis: [paste summary from last session or bootstrap output]

Updates since last session:
- New MEU number: [X]
- Campaigns that ran: [list with results if known]
- Any new data on lifecycle breakdown: [or "no update"]

Task:
1. Update the gap diagnosis with the new data.
2. Identify if the priority lifecycle stage has changed.
3. Flag whether we are on track, at risk, or off track vs. the original target.
4. Recommend any adjustments to the campaign plan.

Output format: same as bootstrap - diagnosis, priority stage, risk flag, recommended adjustments.
```
